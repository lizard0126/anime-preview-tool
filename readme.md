# 🎬 天央动漫社 · 新番导视图片生成工具

> 一个专为 **天央动漫社节操部新番导视活动** 打造的自动化配图工具。
> 多人协作编辑评论，合并生成精美的新番导视图片。

---

## ✨ 功能特性

- **从 yuc.wiki 自动抓取番剧列表** - 输入季度代码（如 `202607`），自动获取动画信息
- **多人协作评论编辑** - 导入/导出评论数据，多人分别添加评论后合并
- **按标题匹配合并** - 即使番剧列表更新，评论也能自动对应到正确的动画
- **实时预览** - 点击"预览"按钮，右侧实时显示生成效果
- **批量导出图片** - 一键导出所有选中的番剧导视图
- **自定义视觉图和头像** - 支持本地图片和网络图片
- **评论图片嵌入** - 评论中可以插入图片，预览时自动嵌入

## 📖 使用指南

### 基础流程

```
1. 获取番剧列表 → 2. 设置默认评论者 → 3. 添加评论 → 4. 预览效果 → 5. 导出图片
```

### 第一步：获取番剧列表

在输入框中输入季度代码（格式：`年份+季度`），点击"从 yuc.wiki 抓取"。

| 季度      | 代码示例   |
| --------- | ---------- |
| 2025年7月 | `202507` |
| 2026年1月 | `202601` |
| 2026年7月 | `202607` |

抓取后会自动保留之前已添加的评论（按标题匹配合并）。

### 第二步：设置默认评论者

在"默认信息"栏输入昵称并选择头像，之后添加评论时会自动填充这些信息。

### 第三步：添加评论

每个动画条目下方都有评论区：

- **添加默认评论** - 一键添加设置了默认昵称和头像的评论
- **添加评论** - 手动输入昵称、头像、奖牌和评论内容
- **插入图片** - 在评论中插入图片，预览时自动显示
- **奖牌标记** - 支持金牌、银牌、黑牌三种奖牌

### 第四步：预览效果

点击动画条目上的"预览"按钮，右侧区域会实时显示生成效果。

### 第五步：导出图片

点击"导出图片"按钮，选择输出目录，工具会自动生成所有选中动画的导视图。

## 👥 多人协作工作流

本项目支持多人分别添加评论后合并，适合社团多人协作。

### 工作流程

```
成员A                 成员B                 负责人
  │                    │                    │
  ├─ 抓取番剧列表      ├─ 抓取番剧列表      ├─ 抓取番剧列表
  ├─ 添加评论          ├─ 添加评论          ├─ 导入所有评论
  ├─ 导出评论 →→→→→→→→→→→→→→→→→→→→→→→→→→→  ├─ 合并评论
  │                    │                    ├─ 导出图片
  └─ 导出评论 →→→→→→→→→→→→→→→→→→→→→→→→→→→  └─ 完成！
```

### 具体步骤

1. **所有成员**各自抓取番剧列表（需要相同的季度代码）
2. **各成员**在对应动画下添加自己的评论
3. **各成员**点击"导出评论"，保存为 JSON 文件
4. **负责人**收集所有 JSON 文件
5. **负责人**点击"导入评论"，选择合并模式
6. **负责人**确认所有评论已合并，导出图片

### 注意事项

- 评论按**动画标题**匹配合并，请确保所有人的番剧列表来自同一季度
- 如果番剧列表更新，已存在的评论会自动保留
- 导入时会根据评论昵称去重，避免同一人的评论重复导入

## ⚠️ 常见问题

### Q: 预览时图片不显示？

A: 检查视觉图和头像文件路径是否正确，支持本地文件和网络图片。

### Q: 导出图片时提示"无法启动浏览器"？

A: 工具需要 Chrome 或 Edge 浏览器。如果未安装，请先安装 Chrome。

### Q: 导入评论后评论没有合并？

A: 检查导入的 JSON 中的动画标题是否与本地数据一致。合并按标题匹配合并。

### Q: 如何保存当前工作进度？

A: 点击"导出评论"保存为 JSON 文件，下次点击"导入评论"恢复。

### Q: 打包后的 exe 文件很大？

A: 因为内嵌了 Node.js 和 Puppeteer，打包后约 200-300MB，属于正常现象。
