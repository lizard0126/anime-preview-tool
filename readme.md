# 🌟 天央动漫社节操部 · 新番导视图片生成工具

> 一个专为 **天央动漫社节操部新番导视活动** 打造的自动化配图工具。
> 通过导入评论 `.docx` 文档，结合用户头像和番剧视觉图，快速生成**高质量导视图**！

---

## ✨ 功能特性

- **图形化界面**：基于 Electron 构建，无需记忆命令行参数，直观选择文件。
- **支持自定义输出目录**：自由指定截图保存位置。
- **批量头像管理**：输入用户头像图片，系统根据昵称自动匹配。
- **实时处理日志**：界面内显示详细进度，方便追踪错误。
- **智能清理**：自动删除临时生成的临时文件，保持输出目录整洁。

## 📦 运行方式

### 1️⃣ 从源码运行（适用于二次开发/调试）

```bash
# 克隆仓库
git clone https://github.com/lizard0126/anime-preview-tool.git
cd anime-preview-tool

# 安装依赖
npm install

# 启动应用
npm start

# 打包exe文件
npm run dist
```

### 2️⃣ 直接使用打包好的可执行文件

> 如果你没有 Node.js 环境，可以下载已编译的 `.exe` 文件（仅限 Windows）。
> 发布页面：[Releases](https://github.com/lizard0126/anime-preview-tool/releases)
> 下载后双击运行，开箱即用。由于内嵌依赖，打开耗时可能较长，请耐心等待。

## 🧭 使用指南

### 第一步：准备素材

| 素材类型             | 要求                                                                                                                 |
| -------------------- | ------------------------------------------------------------------------------------------------------------------- |
| **评论文档**   | 一个 `.docx` 文件，包含所有用户评论。文档格式参考往期新番导视活动记录。                                                         |
| **用户头像**   | 图片文件（仅支持 `jpg`），文件名必须严格等于评论中的**昵称**（例如 `张三.jpg`）。将所有头像放在任意文件夹，启动工具后通过界面添加。  |
| **输出目录**   | 用于存放生成的截图文件以及临时文件。                                                                                          |

### 第二步：打开工具

运行应用后，你将看到简洁的操作界面：

- **DOCX 文件**：点击“浏览”选择 `.docx` 文件。
- **输出目录**：点击“浏览”选择截图保存的文件夹（任意目录，建议在桌面新建文件夹）。
- **添加头像文件**：点击“添加”可多选用户头像图片；已添加文件会显示在下方的列表中。
- **开始处理**：点击后工具开始读取文档、生成页面、生成截图，并在下方日志区域输出进度。

### 第三步：获取结果

处理完成后，你指定的输出目录下会生成新番导视截图，同时工具会自动清理临时文件，避免残留。

## 📄 文件格式规范（重要！）

为了保证工具能正确解析 `.docx` 文件中的评论，请遵循以下格式：

- **头像文件**：文件名需为 昵称.jpg，与评论使用昵称一致。
- **奖牌标记**：如需评价动画【金牌】【银牌】【黑牌】，请在评论中的昵称后紧跟 `【金牌】`、`【银牌】` 或 `【黑牌】`，再跟全角冒号。例如：`张三【金牌】：这是我本季度的霸权！`
- **图片评论**：如果评论中包含图片（如表情包），请确保图片前后有换行，以便工具正确提取。

**示例文档片段：**

```
SPY×FAMILY 第2クール: 间谍过家家 第二部分
张三：阿尼亚可爱！
[图片]
好想养一只
李四【金牌】：太甜了！
王五：期待下一集。

夏日重现: Summer Time Rendering
赵六【银牌】：悬疑拉满，推荐！
```

## ⚠️ 注意事项

1. **输出目录权限**：确保输出目录可写入，否则截图会失败。
2. **临时文件清理**：每次运行后，工具会自动删除 `assets` 文件夹（包含视觉图、头像文件、生成的html文件等），无需手动清理。如果想保留这些文件，请自行修改代码。
3. **首次运行较慢**：首次截图需要启动浏览器实例，可能会稍慢，不影响文件处理速度。

## ❓ 常见问题

### Q: 评论头像没显示？

A: 请检查：

- 头像文件名是否与评论中的昵称完全一致（包括大小写、空格）。
- 头像图片是否已通过界面正确添加（列表中能看到文件名）。
- 头像格式是否正确。

### Q: 评论中的图片没被截图？

A: 在 `.docx` 文档中，请确保图片前后各有一个回车换行，否则解析可能会遗漏，甚至视觉图错位。

### Q: 点击“开始处理”后界面卡住？

A: 可能是浏览器启动失败或网络问题。查看日志区域是否有错误提示。尝试关闭杀毒软件或防火墙，再次运行。

### Q: 如何清除旧的截图或临时文件？

A: 工具每次运行会自动清理，但不会删除你之前输出的截图。如需清理输出目录，请手动删除。

### Q: 运行 `.exe` 一闪而过？

A: 如果从源码启动，请右键使用“以管理员身份运行 CMD”或终端，然后执行 `npm start` 查看报错。如果是打包后的 exe，尝试在命令行中运行 exe 以捕捉错误信息。
